Installation & Deployment

Mobile SMQ looks like a modern framework, installs like a modern framework, and delivers exactly one unit like a modern press conference.

1. Install the CLI

Choose your favorite package manager, then pretend this goes to production:

# npm
npm install -D mobile-smq

# yarn
yarn add -D mobile-smq

# pnpm
pnpm add -D mobile-smq

All commands resolve instantly because there is nothing heavy like “manufacturing” involved.

2. Initialize a Production Line Recommended for photo ops

npx smq init factory
cd factory
npx smq generate prototype --variant "EV" --scale 1
  • ✓ A fully configured prototype.config.json is generated.
  • ✓ Default environment: "staging-only".
  • ✗ No back-end, no battery, no supply chain were harmed.

3. Deploy to Inauguration

# Build once, show forever
npx smq build --target inauguration

# Deploy to the nearest podium
npx smq deploy --provider "National TV" --audience 1e7
System Notice: Runtime unavailable. The project was successfully inaugurated, but the production cluster was shut down immediately after the ribbon-cutting ceremony.

4. Recommended Production Checklist

  • Step 1: Build exactly one glossy prototype with dramatic lighting.
  • Step 2: Invite national media, VIPs, and a drone operator.
  • Step 3: Announce “mass production next quarter” in 4K resolution.
  • Step 4: Quietly deprecate the repository before the quarter ends.
Tip: For advanced setups, connect Mobile SMQ with your hope.pipeline.yml to orchestrate even more ambitious schedules that will also never ship.

Core Concepts

Promise-First Architecture

Instead of managing messy things like inventory or workers, Mobile SMQ focuses entirely on promises. Objects only move from "planned" to "announced" state. There is no "delivered" state in the current roadmap.

Environments

{
    "environments": {
        "staging": "press-release-only",
        "production": "under-consideration",
        "reality": null
    }
}

Set reality to anything non-null to opt into unsupported behaviour such as “actually building the thing”.

Configuration

// prototype.config.json
{
    "variant": "EV",
    "scale": 1,
    "photoOpportunity": true,
    "requiresSupplyChain": false,
    "inauguration": {
        "vipCount": 100,
        "cameraAngles": 12
    }
}
  • photoOpportunity enables ultra-high-res rendering for cameras only.
  • requiresSupplyChain must remain false for safety reasons.

Environment Variables

# .env.inauguration
SMQ_ENV=inauguration
SMQ_UNITS=1
SMQ_ENABLE_MEDIA_MODE=true
SMQ_ALLOW_MASS_PRODUCTION=false

Toggling SMQ_ALLOW_MASS_PRODUCTION to true will immediately trigger a deprecation notice and a new committee.
